## Objective

Install lazydocker TUI for interactive Docker container management from official GitHub releases.

**Purpose:** Provide an intuitive terminal-based interface for managing Docker containers, images, volumes, and networks.
**Output:** 53-lazydocker.sh module installed and executable.

## Context

### Requirements
- LAZY-01: lazydocker installed and available in PATH
- LAZY-02: lazydocker can connect to local Docker daemon

### From CONTEXT.md (Locked Decisions)
- Install from official binary release on GitHub (latest stable)
- Install location: `/opt/lazydocker/` directory
- Create symlink in `/usr/local/bin/lazydocker` for PATH access
- Idempotency: Check if binary exists before downloading
- Tool works with Docker daemon from Phase 6 (assumed ready)

### Codebase Patterns (from 50-docker-engine.sh)
- `set -euo pipefail` at top
- Source lib: `source "$(dirname "$0")/../../lib/common.sh"`
- Idempotency: Check `command -v lazydocker` or check /opt/lazydocker/
- Use `log_info`, `die` from common.sh
- `DEBIAN_FRONTEND=noninteractive` for apt operations

### lazydocker Installation Steps
1. Check if lazydocker already exists (idempotency)
2. Detect system architecture (amd64/arm64)
3. Download latest release from GitHub
4. Extract to /opt/lazydocker/
5. Create symlink: /usr/local/bin/lazydocker -> /opt/lazydocker/lazydocker
6. Verify installation

### GitHub Release URL Pattern
- Repository: jesseduffield/lazydocker
- Release URL: https://github.com/jesseduffield/lazydocker/releases/latest/download/lazydocker_${VERSION}_Linux_${ARCH}.tar.gz
