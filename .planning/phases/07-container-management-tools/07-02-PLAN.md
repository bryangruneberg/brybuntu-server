## Objective

Install ctop for real-time Docker container resource monitoring via apt package manager.

**Purpose:** Provide a terminal-based monitoring tool for viewing container CPU, memory, and resource usage in real-time.
**Output:** 54-ctop.sh module installed and executable.

## Context

### Requirements
- CTOP-01: ctop installed for container resource monitoring
- CTOP-02: ctop displays running containers with CPU/memory metrics

### From CONTEXT.md (Locked Decisions)
- Install via package manager (apt)
- Use `apt-get install ctop` if available in Ubuntu repos
- Fallback to binary release if apt package unavailable
- Tool available for all users immediately after installation
- Works with Docker daemon from Phase 6

### Codebase Patterns (from 50-docker-engine.sh and 51-docker-compose.sh)
- `set -euo pipefail` at top
- Source lib: `source "$(dirname "$0")/../../lib/common.sh"`
- Idempotency: Check with `command -v ctop` or `dpkg -l ctop | grep "^ii"`
- Use `log_info`, `log_warn`, `die` from common.sh
- `DEBIAN_FRONTEND=noninteractive` for apt operations

### ctop Installation Steps
1. Check if ctop already installed via dpkg or command
2. Update apt package list
3. Install ctop via apt-get
4. Verify installation

### Fallback Strategy (if ctop not in apt repos)
1. Check if ctop package available in apt
2. If not, download binary from GitHub (bcicen/ctop)
3. Install to /opt/ctop/ with symlink
