## Test Patterns from Phase 1

From tests/test_10_update.sh and tests/test_20_packages.sh:
- Use @test "description" format
- Use run command for commands that may fail
- Check $status and $output
- Use [[ ]] for assertions
- Tests check for idempotency

Example pattern:
```bash
@test "apt update runs successfully" {
    run bash -c "apt update"
    [[ $status -eq 0 ]]
}
```
