## Security-Critical Patterns from RESEARCH.md

**MANDATORY: visudo -c validation**
- Always validate sudoers syntax BEFORE installing files
- Use temporary file for validation, then atomic mv on success
- Exit code 0 = valid, non-zero = syntax error (would lock out root!)

**MANDATORY: Strict permissions**
- Sudoers.d files MUST be mode 440 (owner read, group read, no write)
- Files must be owned by root:root
- Sudo will refuse to read files with looser permissions

**MANDATORY: Single user per file**
- Each user gets their own file in /etc/sudoers.d/
- Filename matches username: /etc/sudoers.d/bryan
- Content format: `username ALL=(ALL) NOPASSWD: ALL`

## Idempotency Pattern

```bash
# Check if already correct
desired_content="$username ALL=(ALL) NOPASSWD: ALL"
if [[ -f "/etc/sudoers.d/$username" ]] && \
   [[ "$(cat "/etc/sudoers.d/$username")" == "$desired_content" ]]; then
    log_info "Sudoers config for $username already correct, skipping"
    return 0
fi
```

## Syntax Validation Pattern

```bash
# Create temp file for validation
temp_file=$(mktemp)
echo "$rule" > "$temp_file"

# Validate before installing
if ! visudo -c -q -f "$temp_file" 2>/dev/null; then
    rm -f "$temp_file"
    die "Invalid sudoers syntax for $username"
fi

# Only install if validation passed
mv "$temp_file" "/etc/sudoers.d/$username"
chmod 440 "/etc/sudoers.d/$username"
```
